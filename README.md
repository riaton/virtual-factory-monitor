### 1. このリポジトリについて

工場を持たない個人が IT/OT 融合を学ぶための「疑似工場」予知保全検証環境。
`terraform apply` + `docker compose up` で全構成が再現できる。

### 2. 何ができるか？

1. 疑似工場の稼働 — docker compose upで仮想設備machine-01が動き出し、温度・振動・稼働状態の3信号を吐き続ける。周期やノイズの大きさは設定で変えることができる
2. クラウド取り込みと蓄積 — 全信号がX.509証明書付きMQTTでIoT Coreに上がり、ブリッジ経由でInfluxDBに30日分が貯まる。SiteWiseには「横浜工場>ライン>machine-01」というアセットの台帳が立つ
3. 環境の完全な構築と撤収 — terraform applyで立ててdestroyで消すことができる。課金を止めたいとき全撤収できる
4. リアルタイム監視ダッシュボード — Grafanaで3信号のグラフが流れる。ブラウザでlocalhost:3000を開いて見る
5. データ品質の監視 — seqギャップパネルで「欠損がどこで起きたか」、遅延パネル(ingested_at − time)で「データがどれだけ遅れて届いたか」を見ることができる。
6. AIへの日本語での問い合わせ — docker compose run chatで「machine-01、昨夜から様子がおかしくない?」と聞くと、Bedrockが直近データを踏まえて「23時頃から振動RMSが平常の1.8倍で推移しています。温度も緩やかに上昇しており…」と説明する
7. 異常の説明 — 数値の羅列ではなく「何が・いつから・どの程度おかしいか」の文章化。
8. 故障モードの注入 — 操作として、ネットワーク断・重複送信・時刻ズレ・欠損を意図的に起こすことができる。「振動センサーだけ10分間切断」のような実験がコマンドで打てる

## 3. スコープ(作るもの)

```
疑似センサー(Docker) → AWS IoT Core ─→ ブリッジ(購読) → InfluxDB(ローカル) ─→ Grafana(ローカル)
                            └→ SiteWise                              └→ Python CLI → Bedrock
```

| 要素 | 内容 |
|---|---|
| 疑似センサー | Dockerコンテナ × 3系統(温度・振動・稼働状態)。周期・ノイズ・故障モードを設定ファイルで注入可能 |
| 取り込み | AWS IoT Core(MQTT、X.509証明書認証) |
| ブリッジ | IoT Core を MQTT 購読し InfluxDB へ書き込む常駐サービス(IoT Thing として独自証明書を持つ)【ADR-001】 |
| 蓄積 | InfluxDB(ローカル、時系列)+ SiteWise(クラウド、アセットモデル)【ADR-001】 |
| 重複吸収 | InfluxDB の書き込み意味論(同一 measurement+タグ+時刻は同一ポイントへ上書き)【ADR-001】 |
| 可視化 | Grafana — ローカルの docker compose でセルフホスト【ADR-002】 |
| AI層 | Python CLI → InfluxDBクエリ → Bedrock で異常を日本語説明【ADR-003】 |
| IaC | Terraformで全構成をコード化 |
| AI駆動開発 | .claude/(CLAUDE.md・カスタムskill・agents設定)を同梱 |

### 唯一深く作り込む非機能

**ネットワーク断・センサー欠損時の挙動**: デバイス側バッファリング → IoT Core 再送 → InfluxDB の書き込み意味論で重複を吸収。
専用の排除コンポーネントや状態ストアを置かず、重複耐性を**データモデル設計に織り込む**(タグとタイムスタンプの組がそのまま吸収のキーになる。データモデル図に判断コメントとして残す)。
異常系シーケンス図(Mermaid)1本をREADMEに掲載し、シミュレータの故障モード注入(重複送信・時刻ズレ)で「重複を投げても二重計上されない」ことを実演可能にする(詳細はADR-001)。

## 5. 成果物一覧(ドキュメントは5点のみ)

1. アーキテクチャ構成図(README最上部)
2. データモデル図 1枚(InfluxDB の measurement/タグ設計、粒度の判断コメント付き)
3. 異常系シーケンス図 1本(Mermaid、README内)

5. 設計判断の記録(ADR風、含めなかった理由も明記)
